Battery Management System

A first-year engineering project focused on understanding how battery packs can be monitored safely using sensors, embedded programming, and real-time data.

StatusIn progress
AreaEmbedded systems and battery safety
ToolsESP32, MicroPython, sensors
GoalLearn by building and testing

Overview

A Battery Management System is used to monitor and protect a collection of battery cells. It helps track important values such as voltage, current, and temperature during charging and discharging.

This matters because batteries need to stay within safe operating limits. If voltage, current, or temperature move outside a safe range for too long, it can create serious safety issues, especially with lithium-ion batteries.

For this project, I am using the Battery Management System as a way to build practical skills in electronics, embedded programming, data collection, and battery safety.

Why I Chose This Project

I learn best through hands-on work, so building a Battery Management System is a practical way for me to develop real engineering skills while understanding how individual components behave in a larger system.

This project also connects to a longer-term goal: creating a home-built solar power wall with Bluetooth monitoring and data logging to a handheld device. By building the system myself, I can understand the design choices instead of relying only on an off-the-shelf BMS.

What I Am Learning

  • How voltage, current, and temperature are measured in a battery pack.
  • How components such as shunt resistors, voltage dividers, NTC thermistors, and MOSFETs are used in monitoring and protection circuits.
  • How to use an ESP32 with MicroPython for embedded programming.
  • How sensor readings can be collected, interpreted, and used for safer battery operation.
  • How data logging can help with diagnostics and estimating the state of charge.

Battery Safety

Functional safety is one of the most important parts of a Battery Management System. Lithium-ion batteries can be dangerous if they are charged, discharged, or operated incorrectly.

A BMS helps reduce risk by monitoring the battery pack and helping prevent unsafe conditions such as overvoltage, undervoltage, overcurrent, and excessive temperature.

This project is being developed as a learning project. I am treating safety as a core part of the design, not as an extra feature.

System Architecture

The system is based on real-time sensor data from the battery pack. The ESP32 reads sensor values and uses them to monitor the condition of the battery.

  • Voltage sensing: Used to check whether the battery is within a safe voltage range.
  • Current sensing: Used to understand charging and discharging behaviour.
  • Temperature sensing: Used to detect unsafe heating.
  • Control logic: Used to make decisions based on the sensor readings.
  • Data logging: Used to record information for diagnostics and future analysis.

Kalman Filter Interest

One part of this project I am especially interested in is the Kalman filter. It is an algorithm that can combine noisy sensor data, such as ADC readings, with a mathematical model to produce a more reliable estimate.

In this project, I want to explore how a Kalman filter could be used to improve State of Charge estimation. I have not fully implemented this yet, but it is one of the ideas I am keen to investigate as the project develops.
